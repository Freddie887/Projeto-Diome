História da Internet

(voltar)

No final dos anos 50, os EUA formaram uma instituição de investigação, a qual designaram por ARPA.

O Objetivo desta instituição era integrar o Departamento de defesa dos EUA, através de uma rede de comunicações.

Para satisfazer estas necessidades, os dados, eram divididos em pacotes que seriam encaminhados, de forma instantânea, por uma das várias rotas que estivessem disponíveis

Com esta divisão dos dados em pacotes, os diferentes pacotes poderiam seguir caminhos independentes, cujo ponto comum era o destino que levavam.

A Arpanet possibilitou o envio do primeiro E-mail por Ray tomilson somente nos anos 70 que a palavra internet foi usada que era aplicada aos sistemas de redes interconectadas por meios de protocolo TCP/IP

Isso tudo até 1990, quando Tim Berners-Lee, conhecido como o pai da web, cria a linguagem HTML. A equipe do cientista criou o primeiro cliente web “WorldWideWeb” (www).

Visão Geral do HTML

(voltar)

Qual é o principal objetivo do HTML? Sigla para HyperText Markup Language — Linguagem de Marcação de Hipertexto —, o HTML é o componente base da web. Isso quer dizer que ele permite a construção de websites e a inserção de novos conteúdos, como imagens e vídeos, por meio dos hipertextos.

Ele fornece uma estrutura básica para a página, permitindo que os navegadores web saibam como exibir o conteúdo para os usuários. HTML não é usado para estilizar páginas da web ou promover interação ou eventos do usuário, essas tarefas são geralmente realizadas por outras linguagens como CSS e JavaScript.

O que são Tags?

As tags são formadas por uma estrutura própria, iniciam com o sinal “menor que”, em seguida vem o nome daquele elemento e por fim, o sinal “maior que”. Podem ser dispostas em tags que precisam de fechamento e tags que fecham sozinhas (self-closing). O fechamento de uma tag será definido com uma barra (/), sendo que no caso das tags de auto fechamento, não há necessidade da presença desse caractere.

As tags podem ser categorizadas inicialmente em dois tipos, em “nível de bloco” (block-level) e “em linha” (inline). Um elemento em nível de bloco ocupa toda a largura de seu elemento pai, que também chamamos de elemento container, criando assim um “bloco”. Já os elementos inline, geralmente usamos para demarcação de conteúdos de texto.
